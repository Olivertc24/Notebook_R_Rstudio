El poder del Tidyverse: dplyr y tidyr

Prof. Oliver Triveño

¿Qué es el Tidyverse?

  • Es una colección de paquetes diseñados para la Ciencia de Datos.

  • Todos comparten una misma filosofía de diseño, gramática y estructuras de datos.

library(tidyverse) 
# Carga dplyr, tidyr, readr, ggplot2, etc.

Filosofía de Datos “Tidy”

  • La filosofía “tidy” (ordenado) es fundamental en el tidyverse:

    • Cada variable es una columna: Cada tipo de observación que registras debe tener su propia columna.

    • Cada observación (o registro) es una fila: Cada caso o unidad experimental debe ser una fila distinta.

    • Cada unidad de observación debe ser claramente definible: Facilita la comprensión y el análisis.

Ventajas:

  • Código más legible (uso de pipes %>%).

  • Consistencia de funciones: mismos nombres en dplyr, ggplot2, tidyr, readr.

  • Pipeline armonizado: lectura.

El Operador Pipe (%>% o |>) El “pipe” nos permite encadenar operaciones. Se lee como “y entonces…”.

  • %>% (Magrittr): El clásico del tidyverse (Ctrl + Shift + M).
  • |> (Nativo): Disponible desde R 4.1.

Vamos a trabajar con un dataframe de ejemplo:

siniestros <- tibble(
  id = 1:5,
  ramo = c("Auto", "Vida", "Auto", "Salud", "Vida"),
  monto = c(500, 2000, 450, 150, 3000),
  estado = c("Pagado", "Pendiente", "Pagado", "Pagado", "Rechazado")
)

  1. Sin Pipe (Difícil de leer): summarise(group_by(filter(siniestros, monto > 300), ramo))

  2. Con Pipe (Fluido):

siniestros %>% 
  filter(monto > 300) %>% 
  group_by(ramo) %>% 
  summarise(monto)
# A tibble: 4 × 2
# Groups:   ramo [2]
  ramo  monto
  <chr> <dbl>
1 Auto    500
2 Auto    450
3 Vida   2000
4 Vida   3000

2. Los Verbos de dplyr

Select y Filter

  • select(): Selecciona columnas (variables).
  • filter(): Selecciona filas según condiciones.
siniestros %>% 
  filter(monto > 400) %>% 
  select(ramo, monto)
# A tibble: 4 × 2
  ramo  monto
  <chr> <dbl>
1 Auto    500
2 Vida   2000
3 Auto    450
4 Vida   3000

Filtros Múltiples: Operadores Lógicos (& y |)

En la vida real, rara vez filtramos por una sola condición. Para combinar múltiples reglas, usamos los operadores lógicos fundamentales:

  • El operador & (AND / Y): Exige que todas las condiciones se cumplan al mismo tiempo.
  • El operador | (OR / O): Exige que al menos una de las condiciones se cumpla.

Tip

Nota: En dplyr, si separas las condiciones con una coma dentro de filter(), R lo interpreta automáticamente como un &)

Veamos cómo aplicarlos a nuestra tabla de siniestros:

1. Ejemplo con & (Intersección): Supongamos que el área de auditoría nos pide los siniestros del ramo “Auto” Y que ya hayan sido pagados.

siniestros %>% 
  filter(ramo == "Auto" & estado == "Pagado")
# A tibble: 2 × 4
     id ramo  monto estado
  <int> <chr> <dbl> <chr> 
1     1 Auto    500 Pagado
2     3 Auto    450 Pagado

2. Ejemplo con | (Unión): Ahora queremos identificar los siniestros que sean de “Salud” O aquellos cuyo monto sea considerado un impacto alto (mayor a 1000), sin importar el ramo.

siniestros %>% 
  filter(ramo == "Salud" | monto > 1000)
# A tibble: 3 × 4
     id ramo  monto estado   
  <int> <chr> <dbl> <chr>    
1     2 Vida   2000 Pendiente
2     4 Salud   150 Pagado   
3     5 Vida   3000 Rechazado

Tip

El atajo %in%

  • Cuando tienes muchas condiciones | sobre una misma variable (ej. ramo == “Auto” | ramo == “Vida” | ramo == “Salud”), tu código se vuelve largo.

  • El operador %in% hace esto mucho más elegante:

filter(ramo %in% c(“Auto”, “Vida”, “Salud”))

Arrange y Mutate

  • arrange(): Ordena las filas.
  • mutate(): Crea o modifica variables (columnas).
siniestros %>% 
  arrange(desc(monto)) %>% 
  mutate(iva = monto * 0.16,
         total = monto + iva)

Summarise (y Group_by)

La potencia real surge al combinar estos dos. group_by() rompe el dataset en grupos, y summarise() colapsa cada grupo en una sola fila (estadística).

siniestros %>% 
  group_by(ramo) %>% 
  summarise(
    conteo = n(),
    promedio_costo = mean(monto),
    total_ramo = sum(monto)
  )
# A tibble: 3 × 4
  ramo  conteo promedio_costo total_ramo
  <chr>  <int>          <dbl>      <dbl>
1 Auto       2            475        950
2 Salud      1            150        150
3 Vida       2           2500       5000

Extraer filas únicas

3. Reestructurar con tidyr

A veces los datos vienen en formato “Ancho” (Excel) y los necesitamos en formato “Largo” (Tidy) para graficar o modelar.

  • Pivot Longer (De Ancho a Largo)
# Datos "Anchos" (Ej: Reservas por año en columnas)
reservas <- tibble(
  producto = c("Auto", "Vida"),
  res_2023 = c(100, 500),
  res_2024 = c(120, 550)
)
reservas
# A tibble: 2 × 3
  producto res_2023 res_2024
  <chr>       <dbl>    <dbl>
1 Auto          100      120
2 Vida          500      550
  • Pivot Longer (De Ancho a Largo)
# Transformación
reservas_long <- reservas %>% 
  pivot_longer(
    cols = c(res_2023, res_2024),
    names_to = "anio",
    values_to = "reserva"
  )
reservas_long
# A tibble: 4 × 3
  producto anio     reserva
  <chr>    <chr>      <dbl>
1 Auto     res_2023     100
2 Auto     res_2024     120
3 Vida     res_2023     500
4 Vida     res_2024     550
  • Pivot Wider (De Largo a Ancho)

La operación inversa. Útil para presentar reportes finales.

reservas_long %>% 
  pivot_wider(
    names_from = anio,
    values_from = reserva
  )
# A tibble: 2 × 3
  producto res_2023 res_2024
  <chr>       <dbl>    <dbl>
1 Auto          100      120
2 Vida          500      550

4. Unión de Tablas (Joins)

Vincular datos de dos fuentes distintas usando una clave común (key).

polizas <- tibble(id = 1:3, cliente = c("Ana", "Bob", "Cid"))
reclamos <- tibble(id = c(1, 1, 2), monto = c(100, 200, 500))

Left Join Mantiene todas las pólizas, trae reclamos si existen.

left_join(polizas, reclamos, by = "id")
# A tibble: 4 × 3
     id cliente monto
  <dbl> <chr>   <dbl>
1     1 Ana       100
2     1 Ana       200
3     2 Bob       500
4     3 Cid        NA

Inner Join Solo mantiene registros que existen en AMBAS tablas.

inner_join(polizas, reclamos, by = "id")
# A tibble: 3 × 3
     id cliente monto
  <dbl> <chr>   <dbl>
1     1 Ana       100
2     1 Ana       200
3     2 Bob       500
  • Left Join: Todo lo de la izquierda (A).
  • Right Join: Todo lo de la derecha (B).
  • Inner Join: Solo la intersección (\(A \cap B\)).
  • Full Join: Todo (\(A \cup B\)).

5. Manejo de Datos Faltantes (NA)

  • En la vida real, las bases de datos casi nunca están completas, hay errores de sistema o cruces fallidos.

  • Vamos a simular una cartera de 1.000 clientes donde, a propósito, faltan algunos datos.

# Generamos 1000 registros simulados
set.seed(2026) # Para que todos obtengamos los mismos datos aleatorios
n <- 1000

cartera <- tibble(
  poliza_id = 1:n,
  edad = round(rnorm(n, mean = 40, sd = 10)),
  ramo = sample(c("Auto", "Vida", "Salud", "Hogar"), n, replace = TRUE),
  prima = round(runif(n, 500, 3000), 2),
  riesgo = sample(c("Alto", "Medio", "Bajo"), n, replace = TRUE)
)

# Introducimos NA (valores faltantes) de forma aleatoria
cartera <- cartera %>%
  mutate(
    edad = if_else(runif(n) < 0.10, NA_real_, edad),           # ~10% sin edad
    prima = if_else(runif(n) < 0.15, NA_real_, prima),         # ~15% sin prima
    ramo = if_else(runif(n) < 0.05, NA_character_, ramo),      # ~5% sin ramo
    riesgo = if_else(runif(n) < 0.12, NA_character_, riesgo)   # ~12% sin riesgo
  )

head(cartera)

5.2 Estrategia 1: Eliminación (drop_na)

Si los datos faltantes son muy pocos o la variable no es crítica, podemos eliminar las filas incompletas usando la función drop_na() del paquete tidyr.

# Eliminar TODAS las filas que tengan al menos un NA
cartera_limpia <- cartera %>% drop_na()

# Eliminar filas solo si falta la variable 'ramo'
cartera_sin_na_ramo <- cartera %>% drop_na(ramo)

Advertencia

¡Precaución! Usar drop_na() sin argumentos eliminará cualquier fila con un solo NA. En nuestro caso de 1000 registros, podríamos perder más del 30% de la cartera si no tenemos cuidado.

5.3 Estrategia 2: Imputación en Variables Categóricas

En lugar de borrar al cliente que no tiene registrado el ramo o el riesgo, podemos rellenar ese vacío con una categoría como “Desconocido” o “Sin Clasificar”. Usamos replace_na():

cartera <- cartera %>%
  mutate(
    ramo = replace_na(ramo, "No Especificado"),
    riesgo = replace_na(riesgo, "Sin Evaluación")
  )

# Verificamos que ya no hay NA en esas columnas categóricas
colSums(is.na(cartera))
poliza_id      edad      ramo     prima    riesgo 
        0       116         0       140         0

5.4 Estrategia 3: Imputación en Variables Numéricas

Para los números (como la edad o la prima), rellenar con un “cero” arruinaría nuestros promedios y modelos. Una técnica común es imputar la media o la mediana de esa variable.

# Calculamos la mediana de la edad y el promedio de la prima (ignorando los NA)
mediana_edad <- median(cartera$edad, na.rm = TRUE)
promedio_prima <- mean(cartera$prima, na.rm = TRUE)

cartera_imputada <- cartera %>%
  mutate(
    # Si la edad es NA, pon la mediana; si no, deja la edad original
    edad = if_else(is.na(edad), mediana_edad, edad),
    
    # Hacemos lo mismo para la prima
    prima = if_else(is.na(prima), promedio_prima, prima)
  )

# ¡Cartera 100% limpia y sin perder una sola fila!
colSums(is.na(cartera_imputada))
poliza_id      edad      ramo     prima    riesgo 
        0         0         0         0         0
